# 端到端测试

使用无头浏览器对系统的关键用户流程进行自动化测试。

## 触发场景

- 功能开发完毕，需要验证整体流程
- 上线前的最终验收
- 修了一个 Bug，想确认没有破坏其他流程

## 执行策略：多子 agent 并行

1. 先分析项目，识别所有关键用户流程
2. 每个流程启动一个独立的子 agent（使用无头浏览器）
3. 所有流程并行执行，互不干扰
4. 汇总所有结果

## 每个子 agent 的工作

- 启动无头浏览器（Playwright）
- 按流程步骤模拟用户操作
- 每步验证页面状态和数据正确性
- 关键步骤截图留证
- 检查控制台错误和网络请求
- 失败时截图 + 尝试修复 + 重跑（最多 2 轮）

## 输出格式

```
# 端到端测试报告

时间：YYYY-MM-DD HH:MM
测试环境：[URL]

| 流程 | 步骤数 | 状态 | 耗时 | 截图 |
|------|--------|------|------|------|
| 用户登录 | 5 | ✅ | 3s | login.png |
| 创建数据 | 8 | ❌ | 7s | create-fail.png |
| ... | ... | ... | ... | ... |

通过率：X/Y

## 失败详情
1. [流程名] 第 N 步失败
   - 操作：...
   - 期望：...
   - 实际：...
   - 截图：...
```

## 模式参数

- `--quick`：仅测试核心流程（登录 + 最重要的 2-3 个功能）
- `--full`：测试所有已识别的用户流程
- `--url <地址>`：指定测试环境地址
